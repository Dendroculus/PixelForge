# 行为准则

## 我们的承诺

我们希望 PixelForge 成为一个友善、尊重他人且高效的开源项目。

所有参与本项目的人都应保持友善、专业和尊重，无论其背景、经验水平、身份、所在地或在社区中的角色如何。

## 期望行为

积极行为包括但不限于：

- 使用友善且尊重他人的语言
- 提供建设性的反馈
- 对初学者和仍在学习的贡献者保持耐心
- 让讨论聚焦于项目本身
- 尊重不同观点和技术方案
- 以开放态度接受反馈
- 帮助维护协作、安全的社区环境

## 不可接受的行为

不可接受的行为包括但不限于：

- 骚扰、侮辱、歧视或人身攻击
- 恶意挑衅、反复干扰或故意带偏讨论
- 未经允许公开他人的私人信息
- 性暗示语言或不受欢迎的关注
- 威胁、恐吓或敌意行为
- 基于身份、背景或经验水平贬低贡献者
- 任何可能让他人感到不安全或不受欢迎的行为

## 项目标准

参与 PixelForge 时：

- 保持技术讨论尊重且聚焦。
- 批评想法和代码，而不是攻击个人。
- 在可能的情况下假设对方出于善意。
- 避免在聚焦的 pull request 中加入无关改动。
- 尊重 maintainer 对项目方向和范围的决定。

## 报告问题

如果你经历或发现违反本行为准则的行为，请私下联系项目 maintainer。

请尽量提供：

- 发生了什么事情的描述
- 发生位置，例如 issue、pull request、discussion 或消息
- 相关链接或截图
- 是否希望保持匿名

报告会被认真审查，并尽可能以私密方式处理。

## 执行方式

当本行为准则被违反时，maintainer 可以采取适当措施，包括：

- 私下警告
- 要求编辑或删除有害内容
- 临时限制参与项目
- 永久限制参与项目

目标是保护项目社区，并让 PixelForge 保持安全、友善的协作环境。

## 适用范围

本行为准则适用于：

- GitHub issues
- Pull requests
- Discussions
- Code review
- 项目文档
- PixelForge 官方社区空间

## Maintainer 责任

Maintainer 应公平且一致地执行本行为准则，也应以身作则，展示社区所期望的尊重行为。

感谢你帮助 PixelForge 保持友善和专业。
